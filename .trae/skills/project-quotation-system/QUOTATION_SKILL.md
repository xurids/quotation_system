# 工业级分级报价系统设计规范

## 核心原则
- **逻辑绝对严密**：金额必须从叶子节点逐级向上汇总，父级金额 = sum(子级金额)。
- **审计痕迹闭环**：任何偏离原始单价/人月的改动必须高亮并要求填写理由。
- **性能极致丝滑**：针对 500-2000 行的大型树形表格，必须采用按需渲染和异步勾选逻辑。

## 视觉标准
- **Level 0 (系统)**: 18px Bold, Background #F5F7FA, Border-Left 4px Blue.
- **Level 1 (子系统)**: 16px Bold, Background #FAFBFC.
- **Level 2-3 (模块)**: 14px Semi-bold.
- **Leaf (功能点)**: 12px Normal, Editable fields.

## 交互逻辑
- **级联勾选**: 选中父项自动勾选全部子项，计算总价时仅统计勾选的叶子节点。
- **实时汇总**: 底部状态栏始终显示：已选开发费 + 自动规费 = 合同总价。
